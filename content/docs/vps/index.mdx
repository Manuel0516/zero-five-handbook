## Purpose

This handbook catalogs **everything** installed, configured, and running on the Contabo VPS. Like the [MacBook Handbook](/docs/macbook), the goal is **reproducibility**: if the server dies or you migrate to a new provider, these pages let you rebuild the exact same setup step by step.

All state was verified live on the server on 2026-07-11.

## Architecture at a glance

```text
                          Internet
                             │
          zero-five.space (Porkbun → Cloudflare DNS)
                             │
      ┌──────────────────────┼───────────────────────┐
      │ :80 / :443           │ :25565                │ :2020
      ▼                      ▼                       ▼
  Traefik v2.11          Minecraft (Java 17)       sshd
      │                  bare metal, systemd     (key-only)
      │
      ├─ hub.zero-five.space ──→ hub-frontend ─→ hub-backend
      │                                │
      │                                ├─→ docker-socket-proxy → Docker Engine
      │                                └─→ hub-agent :8765 → systemd + RCON
      │
      ├─ ai.zero-five.space ───→ Open WebUI (host :3000)
      │                                └─→ ollama-router (127.0.0.1:11435)
      │                                        ├─→ Ollama VPS    10.8.0.1:11434
      │                                        └─→ Ollama laptop 10.8.0.2:11434
      │                                                  ▲
      │                                       WireGuard tunnel (UDP 51820)
      │
      ├─ brain.zero-five.space → second-brain web → api → PostgreSQL + MinIO
      │
      └─ traefik.zero-five.space → Traefik dashboard (basic auth)

  Telegram translation bot (container) ──→ Ollama VPS 10.8.0.1:11434
```

Three kinds of workloads coexist:

1. **Docker Compose stacks** behind Traefik — the hub, second-brain, and the AI stack. See [Docker](/docs/vps/docker).
2. **Native systemd services** — Ollama, the Minecraft server, and the hub-agent bridge.
3. **A WireGuard VPN** (`wg0`) connecting the VPS (`10.8.0.1`) to the Linux laptop (`10.8.0.2`) so Ollama traffic never touches the public internet. See [WireGuard](/docs/vps/wireguard).

## Rebuilding from scratch

To recreate the server on a fresh Ubuntu VPS, follow the pages in this order:

1. **[Security](/docs/vps/security)** — Create the user, harden SSH, enable UFW and fail2ban first
2. **[Docker](/docs/vps/docker)** — Install Docker Engine and create the external `traefik` network
3. **[WireGuard](/docs/vps/wireguard)** — Bring up the VPN before Ollama, which binds to it
4. **[Ollama](/docs/vps/ollama)** — Install natively, bind to `10.8.0.1`, pull models
5. **[Traefik + Hub](/docs/vps/hub)** — Clone the hub repo; its compose file starts Traefik and the dashboard
6. **[AI Stack](/docs/vps/ai-stack)** — Open WebUI, ollama-router, SearXNG
7. **[Second Brain](/docs/vps/second-brain)** — Clone, configure `.env`, deploy
8. **[Minecraft](/docs/vps/minecraft)** — Restore world backup, install Fabric, enable the unit
9. **[Telegram Bot](/docs/vps/telegram-bot)** — Build and run the container

> **Tip:** DNS (Cloudflare) and the domain survive a server migration — only the A records need to point at the new IP.
