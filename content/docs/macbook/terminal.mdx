The terminal stack is [Ghostty](https://ghostty.org) (emulator) + zsh (shell) + [Starship](https://starship.rs) (prompt), themed with **Catppuccin Mocha** and the **JetBrains Mono Nerd Font**.

---

## Installation

```bash
brew install starship eza bat
brew install --cask font-jetbrains-mono-nerd-font
```

Ghostty is installed as a regular app (download from [ghostty.org](https://ghostty.org)); it lives in `/Applications/Ghostty.app`.

---

## Ghostty configuration

Config file: `~/.config/ghostty/config`

```text
# Appearance
theme = Catppuccin Mocha
background-opacity = 0.72
background-blur = true

# Padding
window-padding-x = 14
window-padding-y = 12
window-padding-balance = true

# Font
font-family = JetBrainsMono Nerd Font
font-size = 15

# Cursor
cursor-style = block
cursor-color = #f5e0dc
cursor-text = #1e1e2e

# macOS
macos-titlebar-style = hidden

# Permissions
clipboard-read = allow
```

The config also pins the full Catppuccin Mocha palette explicitly (`background = #1e1e2e`, `foreground = #cdd6f4`, and `palette = 0..15` entries) so colors stay stable even if the named theme changes upstream.

> **Full reference:** [Ghostty configuration docs](https://ghostty.org/docs/config)

## Zsh configuration

Two files matter:

- `~/.zprofile` ‚Äî Loads Homebrew (`brew shellenv`) and adds `~/.local/bin` to `PATH`
- `~/.zshrc` ‚Äî Conda/mamba init, Starship init, aliases, and the OpenCode `PATH` entry

### Aliases

```bash
alias ls='eza --icons --group-directories-first'
alias ll='eza -lah --icons --group-directories-first'
alias la='eza -la --icons --group-directories-first'
alias cat='bat'
```

- [eza](https://github.com/eza-community/eza) replaces `ls` ‚Äî icons require a Nerd Font
- [bat](https://github.com/sharkdp/bat) replaces `cat` ‚Äî syntax highlighting and paging

### PATH additions

```bash
export PATH="$HOME/.local/bin:$PATH"          # Codex installer, local scripts
export PATH="$HOME/.opencode/bin:$PATH"       # OpenCode CLI
```

### Managed blocks

The `>>> conda initialize >>>` and `>>> mamba initialize >>>` blocks in `~/.zshrc` are generated by `conda init zsh` and `mamba shell init` ‚Äî don't edit them by hand. See [Python](/docs/macbook/python).

## Starship prompt

Config file: `~/.config/starship.toml`

The prompt is a powerline-style bar (Catppuccin colors) showing, in order: username, directory, git branch/status, conda env, language versions (Python, Node, C/C++, Rust, ‚Ä¶), Docker context, and time.

```bash
eval "$(starship init zsh)"   # already in ~/.zshrc
```

> **Tip:** The powerline glyphs (``, ``) require a Nerd Font ‚Äî if the prompt shows boxes, the terminal font is wrong.

> **Full reference:** [Starship configuration docs](https://starship.rs/config/)

## Troubleshooting

### Icons or powerline symbols show as boxes

The terminal is not using a Nerd Font. Check `font-family` in the Ghostty config matches `JetBrainsMono Nerd Font` exactly (run `fc-list | grep -i jetbrains` to confirm the font is installed).

### Prompt looks unstyled in other terminals

Starship only initializes in zsh sessions that source `~/.zshrc`. Non-interactive shells and some IDE-embedded terminals skip it.
