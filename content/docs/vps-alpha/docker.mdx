Docker runs most of the VPS workloads. Unlike the MacBook (which uses [Colima](/docs/macbook/docker)), the VPS runs **Docker CE natively** on Ubuntu.

| Component | Version |
|-----------|---------|
| Docker Engine | 29.6.2 (docker-ce from Docker's apt repo) |
| Compose | v5.3.1 plugin (`docker compose`, not `docker-compose`) |
| Buildx | installed |

The `manuel` user is in the `docker` group, so no `sudo` is needed.

### Install on a fresh server

Follow the [official Docker Engine install for Ubuntu](https://docs.docker.com/engine/install/ubuntu/) (apt repository method), then:

```bash
sudo usermod -aG docker manuel   # log out and back in
docker network create traefik    # the shared external network (see below)
```

## Conventions

### The external `traefik` network

Any container that should be reachable from the internet joins the pre-created external bridge network named `traefik` and declares Traefik labels. Compose files reference it as:

```yaml
networks:
  traefik:
    external: true
```

Backend containers (databases, APIs) stay on stack-internal networks and are never published.

### Docker API boundaries

The Docker socket is equivalent to host-root control when clients can send
powerful API requests. A `:ro` socket mount does not make a Unix socket
read-only. Alpha therefore uses two separate internal proxies:

| Consumer | Allowed API behavior | Lifecycle writes |
|---|---|---|
| Traefik | container/network discovery, events, version and ping GETs | denied (`POST=0`) |
| Hub backend | container discovery, status and log GETs | denied (`POST=0`) |

Container lifecycle requests from the dashboard go to the authenticated
hub-agent instead. That agent checks an exact container-name allowlist and the
three allowed actions before invoking the Docker CLI. Neither proxy can create
containers, change mounts, execute commands, or alter privileges.

### Hub auto-discovery labels

Any container with `hub.*` labels appears automatically on the [dashboard](/docs/vps-alpha/hub) within ~15 seconds:

```yaml
labels:
  hub.enable: "true"
  hub.name: "My App"
  hub.kind: "web"          # web | container | bot
  hub.url: "https://myapp.zero-five.space"
  hub.icon: "🚀"
  hub.visibility: "all"    # all | admin
```

### Host networking for the AI stack

Open WebUI and ollama-router use `network_mode: host` on purpose — they must reach the WireGuard IPs (`10.8.0.1`, `10.8.0.2`), which are host interfaces. Traefik reaches them back through `host.docker.internal` (see [Traefik](/docs/vps-alpha/traefik)).

## Common operations

| Task | Command |
|------|---------|
| Status of everything | `docker ps` |
| Update a stack after `git pull` | `cd <stack dir> && docker compose up -d --build` |
| Restart a stack | `cd <stack dir> && docker compose restart` |
| Follow logs | `docker logs -f <container>` |
| Disk usage | `docker system df` |
| Clean old images | `docker image prune -a` |

> **Tip:** With 8 GB RAM and no swap, the box runs close to full memory. Check with `free -h` before adding new services, and consider stopping the Minecraft server (4–8 GB heap) when it is not in use.

## Backups

Named volumes hold the state worth backing up:

| Volume | Contents |
|--------|----------|
| `hub_hub-data` | Hub SQLite database (users, permissions, audit log) |
| `hub_traefik-certs` | Let's Encrypt certificates (`acme.json`) |
| `secondbrain_postgres-data` | Second-brain PostgreSQL data |
| `secondbrain_minio-data` | Second-brain object storage |
| `server_open-webui` | Open WebUI settings and chat history |

> expired `gdrive:` rclone authorization was removed because Google Drive is not
> part of the chosen design. The owner is building a Raspberry Pi backup target;
> until current encrypted copies and a restore test exist, these volumes have no

The Pi should initiate retrieval over a restricted identity where practical,
and the backup set must be encrypted independently of transport. Include
application-consistent PostgreSQL dumps rather than copying a live database
volume blindly. Define retention, monitor backup age and failures, and restore a
database, an uploaded object, Open WebUI state, and one configuration secret
before calling the system protected.
