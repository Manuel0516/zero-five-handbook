Docker runs most of the VPS workloads. Unlike the MacBook (which uses [Colima](/docs/macbook/docker)), the VPS runs **Docker CE natively** on Ubuntu.

| Component | Version |
|-----------|---------|
| Docker Engine | 29.6.0 (docker-ce from Docker's apt repo) |
| Compose | v2 plugin (`docker compose`, not `docker-compose`) |
| Buildx | installed |

The `manuel` user is in the `docker` group, so no `sudo` is needed.

### Install on a fresh server

Follow the [official Docker Engine install for Ubuntu](https://docs.docker.com/engine/install/ubuntu/) (apt repository method), then:

```bash
sudo usermod -aG docker manuel   # log out and back in
docker network create traefik    # the shared external network (see below)
```

## Conventions

### The external `traefik` network

Any container that should be reachable from the internet joins the pre-created external bridge network named `traefik` and declares Traefik labels. Compose files reference it as:

```yaml
networks:
  traefik:
    external: true
```

Backend containers (databases, APIs) stay on stack-internal networks and are never published.

### Hub auto-discovery labels

Any container with `hub.*` labels appears automatically on the [dashboard](/docs/vps/hub) within ~15 seconds:

```yaml
labels:
  hub.enable: "true"
  hub.name: "My App"
  hub.kind: "web"          # web | container | bot
  hub.url: "https://myapp.zero-five.space"
  hub.icon: "🚀"
  hub.visibility: "all"    # all | admin
```

### Host networking for the AI stack

Open WebUI and ollama-router use `network_mode: host` on purpose — they must reach the WireGuard IPs (`10.8.0.1`, `10.8.0.2`), which are host interfaces. Traefik reaches them back through `host.docker.internal` (see [Traefik](/docs/vps/traefik)).

## Common operations

| Task | Command |
|------|---------|
| Status of everything | `docker ps` |
| Update a stack after `git pull` | `cd <stack dir> && docker compose up -d --build` |
| Restart a stack | `cd <stack dir> && docker compose restart` |
| Follow logs | `docker logs -f <container>` |
| Disk usage | `docker system df` |
| Clean old images | `docker image prune -a` |

> **Tip:** With 8 GB RAM and no swap, the box runs close to full memory. Check with `free -h` before adding new services, and consider stopping the Minecraft server (4–8 GB heap) when it is not in use.

## Backups

Named volumes hold the state worth backing up:

| Volume | Contents |
|--------|----------|
| `hub_hub-data` | Hub SQLite database (users, permissions, audit log) |
| `hub_traefik-certs` | Let's Encrypt certificates (`acme.json`) |
| `secondbrain_postgres-data` | Second-brain PostgreSQL data |
| `secondbrain_minio-data` | Second-brain object storage |
| `open-webui` | Open WebUI settings and chat history |

`~/backups/docker-volumes/` exists for local volume dumps, and `rclone` is configured with a `gdrive:` remote for off-site copies:

```bash
# Example: dump a volume and ship it to Google Drive
docker run --rm -v hub_hub-data:/data -v ~/backups/docker-volumes:/backup \
  alpine tar czf /backup/hub-data-$(date +%F).tar.gz -C /data .
rclone copy ~/backups/docker-volumes gdrive:vps-backups/docker-volumes
```
