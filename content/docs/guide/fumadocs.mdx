## Project Architecture

```
zero-five-handbook/
├── content/docs/          # 📝 All documentation lives here — write MDX files
├── src/
│   ├── app/
│   │   ├── layout.tsx              # Root layout (fonts, provider)
│   │   ├── docs/layout.tsx         # Docs layout (sidebar, search)
│   │   ├── (home)/page.tsx         # Home page
│   │   └── api/
│   │       ├── search/route.ts     # Search API endpoint
│   │       └── chat/route.ts       # AI chat API endpoint
│   ├── components/
│   │   ├── mdx.tsx                 # MDX component provider
│   │   └── markdown.tsx            # Markdown renderer (for LLM export)
│   ├── lib/
│   │   ├── source.ts              # Fumadocs source loader
│   │   └── shared.ts              # Shared route config
│   └── styles/
│       └── themes/                # Theme CSS files
├── source.config.ts               # Fumadocs-MDX collection config
├── opencode.jsonc                  # OpenCode AI provider config
└── package.json
```

### How it works

1. **`source.config.ts`** — Defines the `docs` collection pointing to `content/docs/`
2. **`fumadocs-mdx`** — Processes all `.mdx` files and `meta.json` files at build time, generating a typed collection
3. **`src/lib/source.ts`** — The `loader()` creates a Fumadocs source from the collection, building the page tree
4. **`src/app/docs/layout.tsx`** — Renders the sidebar navigation using the page tree
5. **`src/app/docs/[[...slug]]/page.tsx`** — Renders individual documentation pages

### Collection config (`source.config.ts`)

```ts
import { defineConfig, defineDocs } from 'fumadocs-mdx/config';
import { metaSchema, pageSchema } from 'fumadocs-core/source/schema';

export const docs = defineDocs({
  dir: 'content/docs',
  docs: {
    schema: pageSchema,
    postprocess: {
      includeProcessedMarkdown: true,
    },
  },
  meta: {
    schema: metaSchema,
  },
});

export default defineConfig({
  mdxOptions: {
    // MDX options
  },
});
```

- `includeProcessedMarkdown: true` — Makes rendered markdown available for LLM export (see `src/lib/source.ts`)
- `pageSchema` — The Zod schema for frontmatter (title, description, etc.)
- `metaSchema` — The Zod schema for `meta.json` sidebar config

### Source loader (`src/lib/source.ts`)

```ts
import { docs } from 'collections/server';
import { loader } from 'fumadocs-core/source';
import { lucideIconsPlugin } from 'fumadocs-core/source/lucide-icons';

export const source = loader({
  baseUrl: '/docs',
  source: docs.toFumadocsSource(),
  plugins: [lucideIconsPlugin()],
});
```

The `loader()` builds the page tree. The `lucideIconsPlugin` lets you reference [Lucide icons](https://lucide.dev/icons/) in `meta.json` by name.

## Available Components

All components are imported from `fumadocs-ui/components/*`. Import statements go at the top of your MDX file, after the frontmatter block.

### Components available by default in MDX

These are included in `defaultMdxComponents` (configured in `src/components/mdx.tsx`) and available **without** an explicit import:

- `<Card>` / `<Cards>` — Navigation cards
- `<Callout>` / `<CalloutContainer>` / `<CalloutTitle>` / `<CalloutDescription>` — Callout boxes
- `<CodeBlockTab>` / `<CodeBlockTabs>` — Tabbed code blocks
- `<Heading>` — Anchor-linked headings (used automatically for h1–h6)
- `<Pre>` — Code block wrapper (used automatically for fenced code)
- Smart `<a>` — Internal links automatically use Fumadocs' smart link component
- Smart `<img>` — Images get rounded corners and responsive sizing automatically

### `<Card>` / `<Cards>` — Navigation Cards

Used on landing pages to link to related pages. Automatically available (no import needed).

```mdx
<Cards>
  <Card
    title="Getting Started"
    description="Installation and first steps"
    href="/docs/getting-started"
  />
  <Card
    title="API Reference"
    description="Full API documentation"
    href="/docs/api"
  />
</Cards>
```

Prop reference:

### `<InlineTOC>` — In-page Table of Contents

For long pages that benefit from an on-page outline at the top. Requires import.

```mdx
import { InlineTOC } from 'fumadocs-ui/components/inline-toc';

<InlineTOC
  items={[
    { title: 'Section One', url: '#section-one', depth: 2 },
    { title: 'Section Two', url: '#section-two', depth: 2 },
    { title: 'Subsection', url: '#subsection', depth: 3 },
  ]}
/>
```

| Prop | Type | Description |
|------|------|-------------|
| `items` | `Array<{ title: ReactNode; url: string; depth: number }>` | The visible label, the anchor link (slugified heading ID prefixed with `#`), and its nesting level |

The URL's anchor is the slugified version of your heading text. For example, `## My Section` becomes `#my-section`.

## Adding New Components

Some fumadocs components need to be installed as additional npm packages. Here's how.

### Check if the component exists

First, look in `node_modules/fumadocs-ui/components/` to see what's already available. Most components are already installed with `fumadocs-ui`.

### Install additional packages

If a component requires an external dependency:

```bash
npm install <package-name>
```

For example, `ImageZoom` requires `react-medium-image-zoom` (check if it's already a dependency).

### Add to the MDX provider (optional)

If you want a component to be available project-wide without imports, add it to `src/components/mdx.tsx`:

```ts
import defaultMdxComponents from 'fumadocs-ui/mdx';
import { MyComponent } from 'fumadocs-ui/components/my-component';
import type { MDXComponents } from 'mdx/types';

export function getMDXComponents(components?: MDXComponents) {
  return {
    ...defaultMdxComponents,
    MyComponent,
    ...components,
  } satisfies MDXComponents;
}
```

<Callout type="warn">
  Only add components to the provider if they're used across many pages. For one-off usage, import them directly in the MDX file — it's cleaner and avoids global bloat.
</Callout>

### Add MDX plugins

If you need remark or rehype plugins (like `remark-gfm`), add them to `source.config.ts`:

```ts
export default defineConfig({
  mdxOptions: {
    remarkPlugins: [
      // add remark plugins here
    ],
    rehypePlugins: [
      // add rehype plugins here
    ],
  },
});
```

## Markdown Features

### Basic markdown

| Feature | Syntax |
|---------|--------|
| Bold | `**text**` |
| Italic | `*text*` |
| Code | `` `code` `` |
| Link | `[text](url)` |
| Image | `![alt](src)` |
| Blockquote | `> text` |
| Unordered list | `- item` |
| Ordered list | `1. item` |
| Table | `\| col1 \| col2 \|` |
| Horizontal rule | `---` |

### Code blocks

````markdown
```javascript
console.log('Hello');
```
````

Supported languages: `bash`, `ts`, `tsx`, `js`, `jsx`, `json`, `jsonc`, `yaml`, `mdx`, `css`, `html`, `python`, `go`, `rust`, `sql`, and many more.

### GFM (GitHub Flavored Markdown)

This project includes `remark-gfm`, so these GitHub-specific features work:

- ~~Strikethrough~~ — `~~text~~`
- [x] Task lists — `- [x] done` / `- [ ] todo`
- Tables with alignment — `| :--- | :---: | ---: |`
- Autolinks — `https://example.com` becomes a clickable link
- Footnotes (if configured)

### Math (LaTeX)

Fumadocs supports LaTeX math if the `remark-math` / `rehype-katex` plugins are configured. Inline: `$E = mc^2$` or block:

```latex
$$
\frac{1}{\pi} = \frac{2\sqrt{2}}{9801} \sum_{k=0}^{\infty} \frac{(4k)!(1103+26390k)}{(k!)^4 396^{4k}}
$$
```

## Source API

The `source` object in `src/lib/source.ts` provides the page tree and utilities used by the layouts.

### `source.getPageTree()`

Returns the hierarchical page tree for the sidebar. Used in `src/app/docs/layout.tsx`:

```tsx
<DocsLayout tree={source.getPageTree()} {...baseOptions()}>
```

### `source.getPages()`

Returns a flat list of all pages:

```ts
const pages = source.getPages();
// pages[0].url      -> "/docs/macbook/lama-cpp"
// pages[0].data.title -> "llama.cpp — Local AI Models"
```

### `source.getPage(slugs)`

Gets a single page by its slugs:

```ts
const page = source.getPage(['macbook', 'lama-cpp']);
```

### Source plugins

The project uses the **Lucide Icons plugin**:

```ts
import { lucideIconsPlugin } from 'fumadocs-core/source/lucide-icons';
```

This lets you reference Lucide icon names in `meta.json`:

```json
{
  "title": "MacBook",
  "pages": ["index", "lama-cpp"]
}
```

## Theming and Styling

### Available themes

Theme CSS files live in `src/styles/themes/`:

- `neon.css` — Neon/cyberpunk theme
- `warm.css` — Warm tones theme
- `mix.css` — Mixed theme

### Global styles

Main styles are in `src/app/global.css`. Fumadocs uses Tailwind CSS v4.

### Typography

The project uses JetBrains Mono Nerd Font (loaded locally from `src/app/fonts/`):

```tsx
const jetbrainsNerd = localFont({
  src: [
    { path: './fonts/JetBrainsMono-Regular.ttf', weight: '400' },
    { path: './fonts/JetBrainsMono-Bold.ttf', weight: '700' },

```

### Custom CSS in MDX

You can use `<span className="...">` in MDX files for inline styling:

```mdx
<span className="nerd">󰣇 Linux</span>
```

Custom class names are defined in `global.css` or a theme CSS file.

## Search

This project includes an AI-powered search. The search API is at `/api/search` (defined in `src/app/api/search/route.ts`).

### Using search

Press the **Ask AI** button (floating on docs pages) or use the search bar to find content. The search uses FlexSearch for local indexing and the AI SDK for semantic answers.

### Configuration

Search is configured in `src/app/docs/layout.tsx`:

```tsx
<AISearch>
  <AISearchPanel />
  <AISearchTrigger position="float" className={...}>
    <MessageCircleIcon className="size-4.5" />
    Ask AI
  </AISearchTrigger>
</AISearch>
```

## Troubleshooting

### Page not showing in sidebar

1. Make sure the page file exists in `content/docs/`
2. Add it to the folder's `meta.json` under `pages`
3. If it's in a sub-folder, ensure the parent `meta.json` references the folder name
4. Run `npm run types:check` to regenerate the collections
5. Restart the dev server

### MDX syntax errors

Common issues:

- Unclosed JSX tags: `<Card>` needs `</Card>`
- Invalid frontmatter: YAML must be properly formatted
- Missing imports: Components not in the default provider need explicit `import`
- Wrong code fence: Use triple backticks ` ``` `, not single or double

### Build failing

```bash
npm run build
```

Check the error output. Common fixes:

- Run `npm run types:check` to find type issues
- Run `npx fumadocs-mdx` to regenerate collections manually
- Clear `.next/` cache: `rm -rf .next`

### "Module not found" for a component

The component might not be in the default provider. Check if you need to import it explicitly in your MDX file, or install the package if it's missing.
